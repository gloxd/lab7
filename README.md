# Задание 1

### Текст задачи
В текстовом файле хранятся целые числа по одному в строке. Найти сумму тех элементов файла, которые равны своему индексу (индексацию элементов файла в этой задаче начинать с нуля). Исходный файл необходимо заполнить случайными данными (отдельный метод).
Организовать проверку вводимых значений с клавиатуры. Не использовать автосвойства, LINQ и лямбда-операторы.

### Алгоритм для задания 1
Шаг 1. Безопасный ввод параметров
С клавиатуры запрашивается количество элементов для генерации файла. Ввод защищен циклом с проверкой через `int.TryParse`.

Шаг 2. Генерация исходного файла
Метод `GenerateTask1File` создает текстовый файл и построчно записывает в него случайные целые числа в диапазоне от `0` до `count * 2` с помощью класса `Random`.

Шаг 3. Чтение и подсчет суммы
Метод `RunTask1` открывает файл через `StreamReader`. В цикле `while` построчно считываются элементы. Заводится переменная `index`, которая увеличивается на 1 с каждой итерацией. Если спарсенное число совпадает с текущим значением `index`, оно прибавляется к переменной `sum`.

Шаг 4. Вывод результатов
Программа выводит итоговую сумму элементов, совпавших со своими порядковыми номерами.


# Задание 2

### Текст задачи
В текстовом файле хранятся целые числа по несколько в строке (разделитель — пробел). Вычислить произведение элементов, которые кратны заданному числу k. Исходный файл необходимо заполнить случайными данными (отдельный метод).
Организовать проверку вводимых значений с клавиатуры. Не использовать автосвойства, LINQ и лямбда-операторы.

### Алгоритм для задания 2
Шаг 1. Ввод делителя k
Пользователь вводит с клавиатуры целое число `k`, выступающее делителем. Ввод валидируется на корректность.

Шаг 2. Генерация матрицы чисел
Метод `GenerateTask2File` формирует текстовый файл, состоящий из 5 строк и 4 колонок случайных чисел, разделенных пробелами.

Шаг 3. Поиск и перемножение кратных элементов
Метод `RunTask2` построчно считывает файл через `StreamReader`. Каждая строка разбивается на массив подстрок с помощью метода `String.Split(' ')`. С помощью вложенного цикла `foreach` элементы переводятся в тип `int`. Если элемент делится на `k` без остатка (`number % k == 0`), он умножается на аккумулятор произведения `product`.

Шаг 4. Вывод произведения
Программа выводит на экран вычисленное произведение. Если кратных элементов найдено не было, возвращается 0.


# Задание 3

### Текст задачи
В текстовом файле хранится текст. Переписать в другой файл строки, в которых нет русских букв.
Организовать проверку вводимых значений с клавиатуры. Не использовать автосвойства, LINQ и лямбда-операторы.

### Алгоритм для задания 3
Шаг 1. Создание тестового файла
Программа программно инициализирует входной файл набором строк, содержащих как латиницу, так и кириллицу.

Шаг 2. Построчная фильтрация текста
Метод `RunTask3` параллельно открывает `StreamReader` для исходного файла и `StreamWriter` для файла результатов.

Шаг 3. Поиск русских букв
Для каждой считанной строки запускается цикл `foreach` по её символам. Каждый символ проверяется на вхождение в строку-шаблон, содержащую все строчные и прописные буквы русского алфавита (`абвгде...`). Используется стандартный метод `IndexOf`. Если в строке не найдено ни одного совпадения с русским алфавитом, она записывается в выходной файл.


# Задание 4

### Текст задачи
Бинарные файлы содержат числовые данные. Переписать в другой файл последовательного доступа те элементы, которые кратны k. Исходный файл необходимо заполнить случайными данными (отдельный метод).
Организовать проверку вводимых значений с клавиатуры. Не использовать автосвойства, LINQ и лямбда-операторы.

### Алгоритм для задания 4
Шаг 1. Ввод критерия кратности
С клавиатуры считывается и валидируется целочисленный параметр `k`.

Шаг 2. Запись в бинарный файл
Метод `GenerateTask4BinaryFile` создает файл с помощью `BinaryWriter` и записывает туда поток случайных целых чисел в двоичном формате (`WriteInt32`).

Шаг 3. Фильтрация двоичного потока
Метод `RunTask4` считывает числа из исходного бинарного файла через `BinaryReader` в цикле `while (br.BaseStream.Position < br.BaseStream.Length)`. Элементы, прошедшие проверку на кратность (`number % k == 0`), последовательно записываются во второй файл результатов через новый `BinaryWriter`.


# Задание 5

### Текст задачи
Бинарные файлы содержат величины типа struct/class. Файл содержит сведения об игрушках: название игрушки, ее стоимость в рублях и возрастные границы (например, игрушка может предназначаться для детей от двух до пяти лет). Определить стоимость самого дорогого конструктора. Обязательно использовать XML-сериализацию.
Организовать проверку вводимых значений с клавиатуры. Не использовать автосвойства, LINQ и лямбда-операторы.

### Алгоритм для задания 5
Шаг 1. Описание сущности (Класс Toy)
Создан класс `Toy` с закрытыми приватными полями (`name`, `price`, `minAge`, `maxAge`) и классическими свойствами, содержащими блоки `get` и `set`. Автосвойства полностью исключены.

Шаг 2. Интерактивный ввод массива объектов
Программа запрашивает количество игрушек, после чего в цикле `for` пользователь заполняет текстовые названия и числовые параметры (стоимость, возрастные ограничения) с жесткой валидацией диапазонов.

Шаг 3. XML Сериализация
Метод `RunTask5Save` сериализует полученный массив объектов `Toy[]` в файл `toys.xml` с помощью стандартного класса `XmlSerializer`.

Шаг 4. Десериализация и поиск максимума
Метод `RunTask5LoadAndFind` считывает объекты обратно из файла. Затем с помощью цикла `foreach` отбираются только те элементы, у которых в поле `Name` (приведенном к нижнему регистру) встречается подстрока "конструктор". Среди этих объектов вычисляется максимальное значение поля `Price`.

# Задание 6

### Текст задачи
Составить программу, которая проверяет, есть ли в списке L (коллекция List) хотя бы два одинаковых элемента.
Организовать проверку вводимых значений с клавиатуры. Не использовать автосвойства, LINQ и лямбда-операторы.

### Алгоритм для задания 6
Шаг 1. Динамическое наполнение List
Пользователь указывает желаемый размер коллекции и поочередно вводит целые числа, которые добавляются в стандартный список `List<int>`.

Шаг 2. Проверка на дубликаты методом "знакомства"
Метод `RunTask6` использует два вложенных цикла `for`. Внешний цикл фиксирует текущий элемент по индексу `i`, а внутренний пробегает по всем последующим элементам с индексом `j = i + 1`. Если обнаруживается совпадение (`list[i] == list[j]`), метод немедленно прерывает работу и возвращает `true`. Если циклы завершились без совпадений — возвращается `false`.

# Задание 8

### Текст задачи
Есть перечень музыкальных произведений. Определить для каждого произведения, какие из них нравятся всем n меломанам, какие — некоторым из меломанов, и какие — никому из меломанов (коллекция HashSet).
Организовать проверку вводимых значений с клавиатуры. Не использовать автосвойства, LINQ и лямбда-операторы.

### Алгоритм для задания 8
Шаг 1. Ввод глобального каталога песен
Пользователь вводит с клавиатуры единую строку всех доступных музыкальных произведений, разделенных запятыми. Программа разбивает строку на массив строк и очищает от лишних пробелов.

Шаг 2. Заполнение вкусов меломанов
С клавиатуры считывается количество меломанов `n`. Для каждого меломана создается персональный `HashSet<string>`, в который пользователь вносит названия любимых треков.

Шаг 3. Анализ распределения предпочтений
Метод `RunTask8` перебирает каждую песню из глобального каталога. Во внутреннем цикле проверяется присутствие этой песни в `HashSet` каждого меломана с помощью метода `Contains`. Переменная-счетчик фиксирует количество лайков.

Шаг 4. Классификация множеств
* Если счетчик равен `n`, песня заносится в хеш-сет `likesAll`.
* Если счетчик больше `0`, но меньше `n`, песня заносится в хеш-сет `likesSome`.
* Если счетчик равен `0`, песня уходит в категорию `likesNone`.
Результаты групп выводятся на экран.


# Задание 9

### Текст задачи
Файл содержит текст на русском языке. Напечатать в алфавитном порядке все гласные буквы, которые не входят более чем в одно слово (коллекция HashSet/Dictionary).
Организовать проверку вводимых значений с клавиатуры. Не использовать автосвойства, LINQ и лямбда-операторы.

### Алгоритм для задания 9
Шаг 1. Выделение слов из файла
Текст считывается из файла, переводится в нижний регистр и с помощью метода `String.Split` по массиву знаков препинания разделяется на чистые слова.

Шаг 2. Картирование гласных букв
Создается словарь `Dictionary<char, HashSet<string>>`, где ключами выступают русские гласные буквы, а значениями — хеш-сеты уникальных слов, в которых эти буквы встретились.

Шаг 3. Фильтрация по уникальности
Программа проходит по всем гласным буквам. Если размер привязанного к букве хеш-сета `Count` строго больше нуля и не превышает единицу (`<= 1`), эта буква добавляется в список кандидатов `resultVowels`.

Шаг 4. Алфавитная сортировка
Поскольку использование LINQ запрещено, список символов `resultVowels` сортируется вручную по возрастанию кодов символов с помощью классического алгоритма сортировки пузырьком, после чего выводится на экран.

# Задание 10

### Текст задачи
На вход программе из текстового файла подается информация о результатах олимпиады в формате `<Фамилия> <Имя> <класс> <баллы>`. Победителем становится участник с наивысшим баллом (при условии, что балл > 200), если доля таких лидеров не превышает 20% от общего числа участников. Написать эффективную программу, которая определяет фамилию и имя лучшего участника, не ставшего победителем. Если таких участников несколько, выдается их количество.
Организовать чтение данных из текстового файла. Не использовать автосвойства, LINQ и лямбда-операторы.

### Алгоритм для задания 10
Шаг 1. Описание структуры данных
Объявлена структура `Participant` с закрытыми приватными полями и открытыми свойствами с явными методами доступа доступа `get`/`set`.

Шаг 2. Потоковое чтение и группировка
Программа считывает файл `participants.txt`. Данные преобразуются в массив структур `Participant[]`. Для эффективного подсчета создается коллекция `SortedList<int, int>`, где ключом является набранный балл, а значением — количество людей, получивших этот балл. `SortedList` автоматически ранжирует баллы по возрастанию.

Шаг 3. Определение статуса победителей
Из конца списка `SortedList` извлекается максимальный балл `maxScore` и количество лидеров `winnersCount`. Рассчитывается квота в 20%. Если максимальный балл превышает 200 и число лидеров укладывается в квоту, фиксируется наличие победителей.

Шаг 4. Поиск целевого балла
* Если победители есть, целевым баллом (`targetScore`) становится следующий по величине балл (предпоследний элемент в отсортированном списке уникальных баллов).
* Если победителей нет (из-за превышения квоты или низких результатов), целевым становится сам максимальный балл.

Шаг 5. Анализ и форматированный вывод
Программа берет количество людей `countOfTarget`, набравших целевой балл. Если такой человек один — циклом `foreach` в массиве находится структура участника и печатается строка вида `Фамилия Имя`. Если претендентов несколько — выводится только их суммарное число, что полностью удовлетворяет условию задачи.
